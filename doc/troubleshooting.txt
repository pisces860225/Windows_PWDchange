1. 啟動問題
=========================================

問題：點擊啟動腳本後無反應
解決方案：
- 確認是否以管理員身份運行批處理文件
- 檢查防毒軟體是否阻止了程式啟動
- 驗證 Windows 版本是否為 Windows 10 或 11
- 檢查是否安裝了 .NET Framework 4.8 或更高版本

問題：啟動時出現「端口被佔用」錯誤
解決方案：
- 檢查配置文件中的 port 設置，預設為 5000
- 使用任務管理器查看是否有其他應用程式佔用該端口
- 修改 config.json 中的端口值為其他未使用端口

問題：啟動時出現「無法載入模組」錯誤
解決方案：
- 重新解壓應用程式包確保所有文件完整
- 嘗試在不同位置解壓並運行
- 檢查系統是否符合最低運行要求

=========================================
2. 網頁介面問題
=========================================

問題：瀏覽器頁面顯示「無法連接」
解決方案：
- 確認服務是否已成功啟動（檢查系統托盤圖標是否存在）
- 手動在瀏覽器中輸入 http://localhost:5000（或配置的端口）
- 檢查防火牆是否阻止了本地連接

問題：界面顯示但無法載入用戶列表
解決方案：
- 確認程序是否具有足夠的系統權限
- 檢查日誌文件中的錯誤信息
- 重新啟動應用程式

問題：界面樣式異常或功能按鈕無法點擊
解決方案：
- 清除瀏覽器緩存後重試
- 嘗試使用其他瀏覽器（推薦 Chrome 或 Edge）
- 檢查瀏覽器是否啟用了 JavaScript

=========================================
3. 密碼修改失敗
=========================================

問題：嘗試修改密碼時出現「密碼不符合要求」
解決方案：
- 確保密碼符合複雜度要求：
  * 至少 8 個字符
  * 包含大寫字母、小寫字母、數字、特殊符號中的至少三種
  * 不能包含用戶名
- 檢查密碼兩次輸入是否一致

問題：修改密碼時出現「權限不足」錯誤
解決方案：
- 確認應用程式是以管理員身份運行的
- 檢查目標用戶賬戶是否被鎖定或受到域策略限制
- 嘗試重新啟動應用程式

問題：密碼已更改但無法使用新密碼登錄
解決方案：
- 檢查大小寫鎖定鍵是否開啟
- 確認輸入法是否正確
- 查看日誌確認密碼是否真正成功更改
- 使用「重置用戶密碼」功能再次嘗試

=========================================
4. 系統托盤相關問題
=========================================

問題：找不到系統托盤圖標
解決方案：
- 檢查任務管理器中應用程式是否在運行
- 查看系統托盤是否處於「顯示全部圖標」狀態
- 重新啟動應用程式

問題：系統托盤右鍵菜單無法顯示
解決方案：
- 點擊托盤圖標後稍等片刻再右鍵點擊
- 嘗試先左鍵點擊後再右鍵點擊
- 重新啟動電腦後再試

問題：從系統托盤退出後程序仍在運行
解決方案：
- 在任務管理器中結束進程
- 檢查日誌中是否有相關錯誤信息
- 重新啟動電腦

=========================================
5. 日誌檢查方法
=========================================

當遇到問題時，日誌文件是最重要的診斷工具：

1. 日誌位置：
   - 應用程式目錄下的 logs 文件夾
   - 文件名格式：passwd_YYYY-MM-DD.log

2. 查看日誌的方法：
   - 使用記事本或其他文本編輯器打開日誌文件
   - 通過系統托盤菜單中的「查看日誌」選項
   - 在應用程式界面中點擊「查看日誌」按鈕

3. 日誌中的關鍵錯誤標記：
   - [ERROR] - 嚴重錯誤，需要立即關注
   - [WARNING] - 警告信息，可能導致功能受限
   - [INFO] - 一般信息，通常不是問題所在

4. 保存日誌副本：
   當需要向技術支援報告問題時，請提供完整的日誌文件

6. 連接問題
=========================================

問題：無法從其他設備訪問應用界面
解決方案：
- 確認配置文件中是否允許遠程訪問
- 檢查防火牆設置是否允許該應用的網絡連接
- 確認 config.json 中的 host 是否設置為 "0.0.0.0"

問題：應用界面在某些網絡環境下無法訪問
解決方案：
- 確認網絡設置是否允許本地環回連接
- 檢查是否有安全軟體攔截了連接
- 嘗試使用 IP 地址而非 localhost 進行訪問

=========================================
7. 權限相關問題
=========================================

問題：「無法獲取用戶列表」錯誤
解決方案：
- 確認應用程式是否以管理員身份運行
- 檢查當前用戶是否具有管理員權限
- 在「以管理員身份運行」的命令提示符中運行診斷命令：
  * net localgroup administrators
  * whoami /all

問題：無法修改特定用戶的密碼
解決方案：
- 確認目標用戶是否為系統保留用戶
- 檢查目標用戶是否受到組策略限制
- 查看是否有其他安全軟體或策略干擾

=========================================
8. 常見錯誤代碼解釋
=========================================

錯誤碼: E001
說明: 應用程式無法啟動
解決方法: 檢查程式文件完整性，確認運行環境

錯誤碼: E002
說明: 用戶列表獲取失敗
解決方法: 確認管理員權限，檢查系統用戶管理服務

錯誤碼: E003
說明: 密碼修改操作失敗
解決方法: 檢查用戶狀態和密碼複雜度要求

錯誤碼: E004
說明: 配置文件讀取錯誤
解決方法: 檢查 config.json 是否存在且格式正確

錯誤碼: E005
說明: 網絡端口綁定失敗
解決方法: 確認端口未被佔用，嘗試修改配置使用其他端口

錯誤碼: E006
說明: 系統 API 調用失敗
解決方法: 檢查 Windows 版本兼容性和系統更新狀態
